# Руководство по развертыванию на Timeweb Cloud App Platform

## Подготовка к деплою

### 1. Структура файлов для деплоя

Для успешного развертывания на Timeweb Cloud App Platform в корне репозитория должны находиться:
- `Dockerfile` - инструкция сборки Docker-образа
- `docker-compose.yml` - манифест для управления сервисами

### 2. Особенности платформы Timeweb Cloud

**Важные ограничения:**
- Порты 80 и 443 зарезервированы под веб-сервер платформы
- Проксирование настраивается только для **первого сервиса** в docker-compose.yml
- Нельзя использовать директивы: `volumes`, `cap_add`, `privileged`, `devices` и др.
- Режим сети `host` запрещен

**Наши файлы уже адаптированы под эти требования!**

## Пошаговая инструкция деплоя

### Шаг 1: Подготовка репозитория

1. Убедитесь, что все файлы находятся в корне репозитория:
   ```
   ├── Dockerfile
   ├── docker-compose.yml
   ├── app.py
   ├── requirements.txt
   └── .env.example
   ```

2. Закоммитьте изменения:
   ```bash
   git add Dockerfile docker-compose.yml
   git commit -m "Configure for Timeweb Cloud deployment"
   git push origin main
   ```

### Шаг 2: Создание приложения в панели Timeweb Cloud

1. Перейдите в раздел **App Platform** → **Создать приложение**
2. В типе приложения выберите **Docker** → **Docker Compose**

### Шаг 3: Подключение репозитория

Выберите один из вариантов:

**Вариант A: Из GitHub/GitLab/BitBucket**
- Кликните на название платформы
- Авторизуйтесь в своем аккаунте
- Выберите репозиторий с приложением

**Вариант B: По URL**
- Кликните "Подключите git-репозиторий по URL"
- Вставьте ссылку на ваш репозиторий
- Для приватных репозиториев укажите реквизиты доступа

### Шаг 4: Выбор ветки и коммита

1. Выберите ветку для деплоя (обычно `main` или `master`)
2. Оставьте включенной опцию **"Сборка по последнему выполненному коммиту"** для автодеплоя
   - При каждом `git push` приложение будет автоматически пересобираться

### Шаг 5: Регион и конфигурация сервера

1. Выберите регион размещения (ближайший к вашим пользователям)
2. Выберите конфигурацию сервера:
   - **Минимальная**: 1 CPU, 1 GB RAM (для тестирования)
   - **Рекомендуемая**: 2 CPU, 2 GB RAM (для продакшена)
   - Или настройте произвольную конфигурацию

### Шаг 6: Настройка переменных окружения

Добавьте следующие переменные в настройках приложения:

```bash
POSTGRES_USER=appuser
POSTGRES_PASSWORD=ваш_надежный_пароль
POSTGRES_DB=appdb
PORT=5000
```

**Важно:** Для продакшена используйте сложные пароли!

### Шаг 7: Информация о приложении

1. Задайте имя приложения (например, `fintech-api`)
2. Добавьте комментарий (опционально)
3. Выберите проект для размещения

### Шаг 8: Запуск деплоя

1. Нажмите **"Запустить деплой"**
2. Отслеживайте прогресс во вкладке **"Деплой"**
3. После завершения вы получите:
   - Технический домен вида `ваше-приложение.timeweb.app`
   - SSL-сертификат Let's Encrypt (автоматически)

## Проверка работы

После деплоя проверьте доступность эндпоинтов:

```bash
# Основной эндпоинт
curl https://ваше-приложение.timeweb.app/

# Health check
curl https://ваше-приложение.timeweb.app/health

# Метрики Prometheus
curl https://ваше-приложение.timeweb.app/metrics

# API баланса
curl https://ваше-приложение.timeweb.app/api/balance

# Тест перевода
curl -X POST https://ваше-приложение.timeweb.app/api/transfer \
  -H "Content-Type: application/json" \
  -d '{"amount": 100, "to_account": "ACC-002"}'
```

## Важные замечания

### Работа с базой данных

⚠️ **Внимание!** Данные внутри контейнера не сохраняются при повторном деплое.

**Рекомендации:**
1. Для продакшена используйте **внешнюю облачную БД** (Timeweb Cloud Databases или другой сервис)
2. Обновите `DATABASE_URL` в переменных окружения:
   ```bash
   DATABASE_URL=postgresql://user:password@external-db-host:5432/dbname
   ```

### Мониторинг

Приложение экспортирует метрики Prometheus на эндпоинте `/metrics`. Вы можете:
- Подключить внешнюю систему мониторинга
- Использовать встроенные инструменты Timeweb Cloud

### Автодеплой

При включенном автодплое:
- Каждый `git push` запускает новую сборку
- Старый контейнер удаляется
- Новый контейнер разворачивается с обновленным кодом

### Логи

Логи приложения доступны:
- В панели управления Timeweb Cloud во вкладке "Логи"
- Через API платформы

## Решение проблем

### Ошибка: "Cannot get HTTP 200"

Это предупреждение может появляться, даже если приложение работает. Причины:
- Приложение возвращает 301 редирект вместо 200
- Health check выполняется до полной готовности приложения

**Решение:** Подождите 1-2 минуты, проверьте вручную через curl.

### Ошибка сборки Dockerfile

Проверьте логи деплоя на наличие ошибок:
- Отсутствие зависимостей в `requirements.txt`
- Ошибки синтаксиса в Python-коде
- Проблемы с правами доступа к файлам

### Приложение не отвечает

1. Проверьте логи в панели управления
2. Убедитесь, что приложение слушает порт 5000
3. Проверьте переменные окружения
4. Убедитесь, что healthcheck endpoint `/health` доступен

## Дополнительные настройки

### Использование собственного домена

1. В панели управления перейдите в настройки приложения
2. Добавьте свой домен
3. Настройте DNS записи согласно инструкции Timeweb Cloud
4. SSL-сертификат выпустится автоматически

### Масштабирование

Для увеличения ресурсов:
1. Остановите приложение
2. Измените конфигурацию сервера в настройках
3. Запустите деплой заново

## Файлы конфигурации

### Dockerfile
```dockerfile
FROM python:3.9-slim
RUN apt-get update && apt-get install -y wget
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "app.py"]
```

### docker-compose.yml
```yaml
version: '3.8'
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}
    depends_on:
      db:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "wget --spider -q http://localhost:5000/health || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3
  
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=${POSTGRES_DB}
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
```
