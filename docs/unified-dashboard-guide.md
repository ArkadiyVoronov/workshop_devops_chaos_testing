# 🚀 Объединённый дашборд для Fintech Workshop

Я создал единый дашборд, который объединяет все метрики из предыдущих дашбордов в одном месте.

## 📊 Структура дашборда

### Верхний ряд (0-8 мин)
- **📊 Запросы в секунду (RPS)** - трафик по эндпоинтам
- **⏱️ Latency (P95/P99)** - время ответа с порогами (0.5с жёлтый, 2с красный)

### Второй ряд (8-12 мин)
- **🔥 Error Rate** - процент ошибок 5xx (gauge: 5% жёлтый, 10% красный)
- **💾 Активные транзакции** - текущее количество активных транзакций
- **🔄 Перезапуски контейнера** - счетчик перезапусков приложения

### Третий ряд (12-20 мин)
- **📈 Статусы ответов** - stacked bar chart для 2xx и 5xx
- **📊 Использование памяти приложением** - память контейнера app с порогами

### Четвёртый ряд (20-28 мин)
- **⚙️ Использование CPU** - процент использования CPU приложением
- **🌐 Сетевой трафик** - входящий/исходящий трафик приложения

### Пятый ряд (28-36 мин)
- **🚨 Состояние алертов** - какие алерты в состоянии firing
- **💾 Использование памяти всеми контейнерами** - App, Database, Prometheus, Grafana

## 🔍 Как использовать на кейсах

### Кейс 1: Рост latency (`scripts/run_latency.sh`)
**Что смотреть:**
1. ⏱️ Latency (P95/P99) - резкий рост P95/P99
2. 📊 RPS - может упасть из-за таймаутов
3. 🔥 Error Rate - может вырасти если есть таймауты

**Ожидаемая картина:**
- P95 растёт с ~0.1с до 2-5с
- P99 следует за P95
- RPS может немного упасть
- Error Rate может вырасти до 5-10%

---

### Кейс 2: Рост ошибок (`scripts/run_error_rate.sh`)
**Что смотреть:**
1. 🔥 Error Rate - резкий рост процента ошибок
2. 📈 Статусы ответов - рост 5xx на фоне стабильных 2xx
3. 📊 RPS - общий трафик остаётся стабильным

**Ожидаемая картина:**
- Error Rate растёт до 20-50%
- 5xx бары растут, 2xx остаются стабильными
- Latency может остаться в норме

---

### Кейс 3: Медленная БД (`scripts/run_db_slow.sh`)
**Что смотреть:**
1. ⏱️ Latency (P95/P99) - рост времени ответа
2. 💾 Активные транзакции - увеличение количества
3. 📊 RPS - может упасть из-за блокировок

**Ожидаемая картина:**
- Latency растёт постепенно
- Active Transactions увеличивается
- RPS падает из-за медленных запросов

---

### Кейс 4: Утечка памяти (`scripts/run_memory_leak.sh`)
**Что смотреть:**
1. 📊 Использование памяти приложением - постоянный рост
2. 🔄 Перезапуски контейнера - увеличение счетчика
3. ⚙️ Использование CPU - может расти из-за GC

**Ожидаемая картина:**
- Memory usage растёт линейно
- При достижении лимита - контейнер перезапускается
- Restarts counter увеличивается

---

### Кейс 5: Хаос микс (`scripts/run_chaos_mix.sh`)
**Что смотреть:**
- **Все панели одновременно!**
- Корреляция между метриками
- Какие метрики реагируют первыми

**Ожидаемая картина:**
- Комбинация всех вышеперечисленных эффектов
- Важно определить primary signal vs secondary effects

---

## 🎯 Преимущества объединённого дашборда

1. **Единая картина** - все метрики в одном месте
2. **Корреляция событий** - легко видеть взаимосвязи
3. **Быстрый триаж** - не нужно переключаться между дашбордами
4. **Экономия места** - компактное расположение панелей
5. **Легче для демонстрации** - один экран показывает всё

## 📁 Расположение файлов

- **Основной файл:** `/workspace/config/grafana/dashboards/unified-dashboard.json`
- **Для provisioning:** `/workspace/config/grafana/provisioning/dashboards/unified-dashboard.json`
- **UID дашборда:** `fintech-unified-dashboard`

## 🔄 Как обновить в Grafana

После запуска docker-compose:
```bash
docker-compose restart grafana
```

Grafana автоматически подхватит новый дашборд через provisioning.

## 🧪 Запуск тестовых кейсов

```bash
# Запустить кейс latency
bash scripts/run_latency.sh

# Запустить кейс error rate
bash scripts/run_error_rate.sh

# Запустить кейс медленной БД
bash scripts/run_db_slow.sh

# Запустить кейс утечки памяти
bash scripts/run_memory_leak.sh

# Запустить микс хаоса
bash scripts/run_chaos_mix.sh
```

## 📝 Заметки по настройке порогов

В дашборде установлены следующие пороги:
- **Latency:** 0.5с (warning), 2с (critical)
- **Error Rate:** 5% (warning), 10% (critical)
- **Memory:** 300MB (warning), 450MB (critical)
- **CPU:** 70% (warning), 90% (critical)

Можно настроить под ваши SLO!
