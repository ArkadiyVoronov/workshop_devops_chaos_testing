# 📊 Скриншоты и данные для кейсов

Этот файл содержит JSON-дампы метрик для каждого кейса. Используйте их для:
- Создания скриншотов "до/после" в Grafana
- Документирования ожидаемого поведения метрик
- Тестирования дашбордов

---

## 🔧 Как сделать скриншот

1. Откройте Grafana: http://72.56.8.118:3000
2. Перейдите на дашборд **Unified Dashboard**
3. Запустите кейс (см. ниже)
4. Подождите 15-30 секунд
5. Сделайте скриншот нужной панели

---

## 📈 Кейс 1: Latency Spike

**Запуск:**
```bash
bash scripts/01_run_latency.sh
```

**Ожидания:**
- P95 latency: ~0.05 сек → **~2.4 сек**
- RPS: без изменений
- Error Rate: 0%

**Метрики (baseline → during):**
```json
// Baseline P95 latency
{"value": ["timestamp", "0.05"]}

// During latency injection
{"value": ["timestamp", "2.4"]}
```

**Что скриншотить:**
- Панель "⏱️ Latency (P95/P99)" — резкий рост графика
- Панель "📈 Запросы в секунду (RPS)" — без изменений (доказывает, что проблема не в нагрузке)

---

## 🔥 Кейс 2: Error Rate Spike

**Запуск:**
```bash
bash scripts/02_run_error_rate.sh
```

**Ожидания:**
- Error Rate: 0% → **~50%**
- RPS: без изменений (но 50% ответов — 500)
- Latency: без изменений

**Метрики:**
```json
// Error Rate during injection
{"value": ["timestamp", "50"]}
```

**Что скриншотить:**
- Панель "🔥 Error Rate" — gauge показывает ~50%
- Панель "📈 Статусы ответов" — красные бары 5xx рядом с зелёными 2xx

---

## 🐢 Кейс 3: Slow Database

**Запуск:**
```bash
bash scripts/03_run_db_slow.sh
```

**Ожидания:**
- Active Transactions: рост (очередь в БД)
- Latency: умеренный рост
- RPS: может упасть

**Метрики:**
```json
// Active DB transactions
{"value": ["timestamp", "0"]}  // baseline
{"value": ["timestamp", "10+"]}  // during
```

**Что скриншотить:**
- Панель "💾 Активные транзакции" — рост
- Панель "⏱️ Latency (P95/P99)" — умеренный рост

---

## 💾 Кейс 4: Memory Leak

**Запуск:**
```bash
bash scripts/04_run_memory_leak.sh
```

**Ожидания:**
- Memory Usage: постепенный рост
- Container Restarts: может вырасти при OOM
- Latency: может расти из-за GC

**Метрики:**
```json
// Container memory usage
{"value": ["timestamp", "150000000"]}  // 150MB baseline
{"value": ["timestamp", "450000000+"]}  // 450MB+ during
```

**Что скриншотить:**
- Панель "📊 Использование памяти приложением" — восходящий тренд
- Панель "🔄 Перезапуски контейнера" — если был OOM kill

---

## 🌪️ Кейс 5: Chaos Mix

**Запуск:**
```bash
bash scripts/05_run_chaos_mix.sh
```

**Ожидания:**
- Все метрики одновременно:
  - Latency: высокий
  - Error Rate: высокий
  - Memory: растёт
  - Transactions: рост

**Что скриншотить:**
- **Весь дашборд целиком** — показывает комплексный инцидент
- Панель "🚨 Состояние алертов" — должны сработать алерты

## 📸 Идеи для скриншотов

### Для презентации/документации:

1. **Baseline (здоровая система)**
   - Все метрики в норме
   - Зелёные индикаторы

2. **Latency кейс**
   - До: P95 < 100ms
   - После: P95 > 2 сек

3. **Error Rate кейс**
   - До: 0% ошибок
   - После: 50% ошибок

4. **DB Slow кейс**
   - До: 0 активных транзакций
   - После: растущая очередь

5. **Memory Leak кейс**
   - До: стабильное использование памяти
   - После: линейный рост

6. **Chaos Mix**
   - Все панели в красной зоне одновременно

### Для обучающих материалов:

- **Алерты**: скриншот панели "🚨 Состояние алертов" когда сработал алерт
- **Target Health**: скриншот "🎯 Target Health" если сервис упал
- **Сравнение**: два скриншота рядом (до/после) для каждого кейса

## 🛠️ Автоматизация скриншотов

Если нужен автоматический скриншотинг, используйте:

```bash
# Установить scrot
apt-get install -y scrot

# Сделать скриншот
scrot /tmp/grafana-screenshot.png

# С задержкой (открыть Grafana, затем запустить)
scrot -d 5 /tmp/grafana-screenshot.png
```

Для headless-режима (без GUI) используйте API Grafana для экспорта панелей:

```bash
# Экспорт панели как PNG (требует авторизации)
curl -u admin:password \
  "http://72.56.8.118:3000/render/d-solo/unified-dashboard/1?panelId=1&width=1000&height=500" \
  -o panel1.png
```
